# PUBG 随机武器挑战 — 完整项目文档

## 一、项目概述

这是一个面向PUBG四人组队的娱乐工具Web应用。核心玩法：每局游戏前，4名玩家通过随机抽取获得限定武器+瞄准镜，只能使用抽到的装备进行游戏，游戏结束后根据伤害尾数+随机运算计分。配合特殊事件卡和惩罚卡增加策略性和娱乐性。

纯前端实现，无框架无后端，直接打开index.html即可运行。

## 二、技术架构

```
index.html              单页应用入口，8个视图section通过JS切换显示
css/style.css           全局样式，深蓝色暗色主题
js/data.js              游戏数据：武器库、地图、运算池、卡牌池
js/wheel.js             武器抽取动画（老虎机式文字滚动）
js/cards.js             卡牌渲染和交互
js/scoring.js           计分引擎和结算动画
js/app.js               主控逻辑、状态管理、页面流程
```

加载顺序：data → wheel → cards → scoring → app。全部通过全局变量交互（App、WeaponDraw、Cards、Scoring）。状态存localStorage防刷新丢失。

## 三、游戏流程

```
设置页（输入昵称、选地图、选局数/转盘模式）
  ↓
首轮抽卡（每人翻牌抽1张特殊事件卡，可保留）
  ↓
每局循环：
  ├─ 使用卡牌（可选，从手牌中选择使用）
  ├─ 抽武器（每人抽2把，老虎机动画 或 同时随机）
  ├─ 抽瞄准镜（每人随机1个倍镜）
  ├─ 装备总览 → 进入PUBG游戏
  ├─ 结算页：输入伤害 → 计分动画 → 排名
  ├─ 第1名和第4名各抽1张事件卡
  └─ 每3局结束：阶段垫底者抽惩罚卡
  ↓
总局数打完 → 终局排名页（总分排名+每局明细表）
```

权重越大越容易被抽到。空投武器权重极低(0.3)。

| 等级 | 计分系数 | 典型权重 | 定位 |
|------|---------|---------|------|
| S | ×0.8 | 0.3~1 | 空投/版本强势，极难抽到 |
| A | ×1.0 | 3 | 主流可用 |
| B | ×1.3 | 7 | 非主流 |
| C | ×1.8 | 9 | 整活级别，高补偿 |

### 去重机制

已被其他玩家抽到的武器，权重降为原来的15%（被抽2次降为2.25%），不完全排除但大幅降低重复。

### 瞄准镜随机池

抽完武器后每人随机1个倍镜，只能用这个倍镜。在游戏里捡不到就只能机瞄。

| 瞄准镜 | 权重 |
|--------|------|

| 2倍镜 | 25 |
| 3倍镜 | 20 |
| 4倍镜 | 15 |
| 6倍镜 | 10 |
| 8倍镜 | 4 |
| 15倍镜 | 1 |

## 五、计分系统

### 计分流程

```
伤害尾数(个位数) → 按武器类型随机运算 → 乘以等级系数 → 取整
两把武器各算一遍 → 取较高分 → 应用卡牌效果 → 最终得分(整数)
```

### 各武器类型运算池

| 类型 | 风格 | 运算(概率) |
|------|------|-----------|
| AR | 均衡 | ×1(30%), +2(20%), -2(20%), ×2(15%), ×0(15%) |
| DMR | 稳定 | ×1(35%), +3(25%), +1(20%), -1(10%), ×2(10%) |
| SMG | 赌狗 | ×3(15%), ×2(20%), ×0(25%), +5(15%), -3(25%) |
| SR | 极端 | 尾数²(20%), ×3(15%), ×0(30%), +7(15%), ×1(20%) |
| SG | 混沌 | ×4(10%), ×0(30%), 尾数²(15%), +10(5%), -5(20%), ×1(20%) |

减法运算最低为0。

### 计分示例

> 伤害287 → 尾数7
> 主武器VSS(C档DMR)：随机到+3 → 7+3=10 → ×1.8(C档) = 18分
> 副武器S686(C档SG)：随机到×0 → 0 → ×1.8 = 0分
> 取高分 → 18分

## 六、特殊事件卡

### 获取时机
- 第1局开始前：每人免费抽1张
- 每局结束后：本局第1名和第4名各抽1张
- 每人最多持有2张

### 使用时机
- 每局回合页面点"使用卡牌"按钮
- 部分卡牌需要选择目标玩家

### 全部卡牌

**武器类：**

| 卡牌 | 效果 | 权重 |
|------|------|------|
| 拳头哥 👊 | 指定一人本局只能用拳头，计分×5补偿 | 8 |

## 七、惩罚系统

### 触发条件
- 每3局结算一次（第3局后、第5局后……）
- 该阶段总分垫底的人抽1张惩罚卡
- 效果持续下一局
- 可花费当前总积分的20%重抽一次

### 全部惩罚卡

| 惩罚 | 效果 |
|------|------|
| 蹲行者 🦆 | 全程只能蹲着移动 |
| 静步大师 🤫 | 全程只能静步 |
| 螃蟹步 🦀 | 只能侧身移动（只按A/D） |
| 跳跳虎 🐯 | 移动时每3秒必须跳一次 |
| 趴地哥 🐍 | 前20人后只能趴着打 |
| 开镜行者 🔫 | 全程开镜移动 |
| 盲狙 🙈 | 不能开镜，只能腰射 |
| 裸眼战士 👁️ | 不能捡倍镜，只用机瞄 |
| 裸奔 🩲 | 不能穿防弹衣和头盔 |
| 极简主义 🎒 | 只能用1级包 |
| 和平主义者 ☮️ | 前5分钟不能开枪 |
| 独狼 🐺 | 不能与队友同乘载具 |
| 步行者 🚶 | 前3分钟不能使用载具 |

## 八、赛制结构

- 默认5局制（可选3/5/7局）
- 转盘模式可选：轮流转（依次动画）/ 同时转（一键全随机）
- 每3局触发一次惩罚结算
- 最后一局结束后显示总分排名和每局明细

## 九、UI/交互设计

- 深蓝色暗色主题（#0f0f1a背景，#4361ee主题蓝）
- 单页应用，8个view通过添加/移除active class切换
- 武器抽取：老虎机式文字快速滚动减速停止（非Canvas转盘）
- 卡牌：翻牌动画，card-back → card-front
- 计分结算：逐个玩家依次展示运算过程动画
- 弹窗：模态框，毛玻璃遮罩，点击外部关闭
- 响应式：支持手机端（@media max-width 600px）

## 十一、已知设计决策

1. 空投武器(Groza/FAMAS/MK14/AWM/P90/Lynx)权重极低(0.3~0.5)，Beryl/M416虽为S档但权重1（地面强势枪）
2. AUG不是空投武器，归为A档
3. 两把武器允许同类型（可以两把AR），但去重机制降低同枪概率
4. 计分结果为整数，无小数
5. 惩罚全部是游戏内操作限制，不涉及灵敏度调整或开麦等游戏外行为
6. 卡牌需要目标有武器时才能使用weapon类卡牌（如升级/降级/偷梁换柱），否则弹窗提示
